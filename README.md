# Измерительный модуль грунтовых вод

> Проект по созданию системы мониторинга грунтовых вод с визуализацией данных в реальном времени.

## Описание проекта

Разработана система для измерения и мониторинга параметров грунтовых вод. Устройство собирает данные с четырёх датчиков, отображает их на LCD-дисплее, сохраняет в базу данных и предоставляет веб-интерфейс с графиками в реальном времени.

### Измеряемые параметры

| Параметр | Датчик |
|----------|--------|
| Уровень воды | HC-SR04 |
| Температура воды | DS18B20 |
| Температура воздуха | DHT22 |
| Влажность воздуха | DHT22 |

## Технологии

### Аппаратная часть (SimulIDE)

| Компонент | Модель | Назначение |
|-----------|--------|------------|
| Микроконтроллер | Arduino Uno | Управление датчиками и обработка данных |
| Ультразвуковой датчик | HC-SR04 | Измерение расстояния до воды |
| Датчик температуры/влажности | DHT22 | Измерение температуры и влажности воздуха |
| Водонепроницаемый датчик | DS18B20 | Измерение температуры воды |
| Дисплей | LCD 16x2 (I2C) | Локальное отображение данных |
| Виртуальный COM-порт | Serial Port (SimulIDE) | Передача данных на ПК |

### Программная часть

| Компонент | Технология             | Назначение |
|-----------|------------------------|------------|
| Прошивка | C/C++ (Arduino)        | Чтение датчиков, вывод на LCD, отправка данных |
| Бэкенд | Python + Flask         | API сервер, обработка данных |
| База данных | SQLite                 | Хранение истории измерений |
| Фронтенд | HTML/CSS/JS + Chart.js | Визуализация данных, графики |

## Схема подключения датчиков

### Arduino Uno → Датчики

| Arduino Pin | Компонент | Вывод компонента |
|-------------|-----------|------------------|
| 9 | HC-SR04 | TRIG |
| 10 | HC-SR04 | ECHO |
| 6 | DHT22 | DATA |
| 2 | DS18B20 | DQ |
| A4 | LCD I2C | SDA |
| A5 | LCD I2C | SCL |

### Дополнительные компоненты

- **Резистор 4.7 кОм** между Pin 2 (DS18B20 DQ) и 5V
- **Потенциометр 10 кОм** на вход `In` HC-SR04 для имитации уровня воды

## Установка и запуск Python-сервера

### Настройка COM-порта

1. Установи **com0com** (виртуальный null-модем)
2. Создай пару портов, например `COM10` ↔ `COM11`
3. В SimulIDE настрой **Serial Port** на `COM11`, скорость 9600
4. В `app.py` укажи порт:
   ```python
   SERIAL_PORT = 'COM10'   # Второй порт из пары
   ```

### Запуск

```bash
python app.py
```

### Открытие веб-интерфейса

Перейди в браузере: **http://127.0.0.1:5000**

---

## Веб-интерфейс

### Функциональность

- **4 карточки** с текущими значениями всех параметров
- **4 отдельных графика** для каждого измеряемого параметра
- Автоматическое обновление каждые 2 секунды

### API Endpoint

`GET /api/data` возвращает JSON с последними 50 измерениями:

```json
{
  "labels": ["12:00", "12:02", ...],
  "levels": [200, 198, ...],
  "humidities": [73.5, 73.2, ...],
  "water_temps": [21.0, 21.2, ...],
  "air_temps": [19.5, 19.6, ...],
  "latest_level": 200,
  "latest_humidity": 73.5,
  "latest_water_temp": 21.0,
  "latest_air_temp": 19.5
}
```
